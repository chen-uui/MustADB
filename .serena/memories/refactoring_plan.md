# 代码重构计划

## 阶段1: 清理重复文件 (高优先级)

### 1.1 移除备份文件
- **删除**: `views_qa_backup.py` - 明显的备份文件，应该被移除
- **验证**: 确认没有其他地方引用这个文件

### 1.2 合并重复的视图文件
- **保留**: `views_qa.py` 作为主要的问答视图文件
- **合并**: 将 `views_qa_extraction.py` 中的数据提取功能迁移到专门的提取视图文件
- **删除**: 合并完成后删除 `views_qa_extraction.py`

## 阶段2: 统一服务层 (高优先级)

### 2.1 创建统一的服务管理模块
- **创建**: `services/__init__.py` 
- **移动**: 将 `get_qa_service()` 函数移到统一的服务模块
- **更新**: 所有引用该函数的文件导入路径

### 2.2 重构RAGService导入
- **标准化**: 所有文件使用统一的导入方式
- **集中管理**: 通过服务管理模块统一管理RAGService实例

## 阶段3: 提取公共组件 (中优先级)

### 3.1 创建基础视图类
- **创建**: `base_views.py` 包含公共的视图逻辑
- **提取**: 将 `_stream_response` 等公共方法移到基类

### 3.2 创建工具模块
- **创建**: `utils/response_utils.py` 包含响应处理工具
- **创建**: `utils/stream_utils.py` 包含流式响应工具

## 阶段4: 优化文件结构 (中优先级)

### 4.1 按功能重组文件
```
pdf_processor/
├── services/
│   ├── __init__.py          # 服务管理
│   ├── rag_service.py       # RAG服务
│   └── qa_service.py        # 问答服务
├── views/
│   ├── __init__.py
│   ├── base_views.py        # 基础视图类
│   ├── qa_views.py          # 问答视图
│   └── extraction_views.py  # 数据提取视图
├── utils/
│   ├── __init__.py
│   ├── response_utils.py    # 响应工具
│   └── stream_utils.py      # 流式工具
└── management/
    └── commands/            # 管理命令
```

### 4.2 清理管理命令
- **合并**: 功能重叠的管理命令
- **重命名**: 使命令名称更清晰
- **文档**: 为每个命令添加清晰的文档

## 阶段5: 依赖关系优化 (低优先级)

### 5.1 分析循环依赖
- **检查**: 使用工具检测循环依赖
- **重构**: 解决发现的循环依赖问题

### 5.2 优化导入结构
- **标准化**: 统一导入顺序和风格
- **最小化**: 减少不必要的导入

## 实施建议

### 风险控制
1. **备份**: 在开始重构前创建完整备份
2. **测试**: 每个阶段完成后运行完整测试
3. **渐进式**: 一次只重构一个模块，避免大规模改动

### 测试策略
1. **单元测试**: 为重构的模块添加单元测试
2. **集成测试**: 确保模块间交互正常
3. **回归测试**: 确保现有功能不受影响

### 优先级说明
- **高优先级**: 直接影响开发效率和代码维护的问题
- **中优先级**: 改善代码结构和可读性的改进
- **低优先级**: 长期代码质量改进

这个重构计划应该能显著减少代码冗余，提高维护性，并减少AI修改代码时的错误率。

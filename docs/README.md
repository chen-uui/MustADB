## 🎯 核心问题

### 当前系统问题
1. **服务层冗余**: 4个相似的服务类
2. **API接口过多**: 50+个接口，参数不一致
3. **搜索逻辑复杂**: 三级关键词搜索，性能差
4. **配置管理混乱**: 8个配置文件，选项过多
5. **错误处理分散**: 每个服务独立处理错误

### 简化目标
1. **统一服务**: 合并为1个UnifiedRAGService
2. **简化API**: 减少到4个核心接口
3. **优化搜索**: 单一搜索策略，提升性能
4. **统一配置**: 2个配置文件，合理默认值
5. **统一错误处理**: 集中式异常管理

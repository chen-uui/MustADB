# End-to-End LLM-Only vs RAG Baseline Evaluation Design

## Purpose

This experiment tests whether retrieval evidence improves answer generation when the answer-generation model is held constant. It is designed for the reviewer question: whether a large LLM can replace the retrieval/evidence-grounding component.

The comparison is model-agnostic: a larger or smaller local LLM may be used as the generation backend, but each condition must use the same generation model and decoding settings.

## Pipelines

1. `llm_only`: question and task instruction only; no retrieved evidence.
2. `bm25_rag`: BM25 ranked evidence plus the same local LLM.
3. `dense_rag`: dense retrieval ranked evidence plus the same local LLM.
4. `hybrid_rag`: hybrid retrieval ranked evidence plus the same local LLM.
5. `hybrid_rerank_rag`: hybrid candidates reranked by `cross-encoder/ms-marco-MiniLM-L-6-v2`, then passed to the same local LLM.

The qrels annotation template is used only as a metadata/snippet cache. Its relevance fields are not used as answer-level labels.

Therefore this baseline reuses the 20 unique questions and creates 100 prompt inputs: 20 questions x 5 pipelines.

## Evidence Policy

- No corpus-wide context or full PDFs are supplied to the LLM.
- RAG prompts include only ranked top-k evidence snippets.
- Default prompt-preparation setting: `evidence_top_k = 5`.
- The script supports changing this value with `--evidence-top-k`.
- Missing metadata or snippets remain blank/`NA`; they are not fabricated.

## Evaluation Plan

After answers are generated by the same local LLM for all pipelines, annotate each answer using the labels in `evaluation_guidelines.md`.

Primary metrics by pipeline:

- fully supported count / percentage
- partially supported count / percentage
- unsupported count / percentage
- evidence mismatch count / percentage
- answer correctness / accuracy, only if gold answers are supplied

Do not use retrieval qrels as answer-level support labels. Retrieval qrels measure document relevance; this experiment measures generated-answer correctness and evidence support.
